    Cilium BGP 控制平面由一组自定义资源管理，这些资源提供了一种灵活的方式来配置 BGP 对等体、策略和通告。

    • CiliumBGPClusterConfig：定义应用于多个节点的 BGP 实例和对等配置。设置对等端点，并且可以引用一个或多个 CiliumBGPPeerConfig 资源（使用资源名称） ，
    • CiliumBGPPeerConfig：一套通用的BGP对等连接配置。它可以用于多个对等体之间。 配置对等互连的行为方式，并且可以引用一个或多个 CiliumBGPAdvertisement 资源（使用选择器）
    • CiliumBGPAdvertisement：指定应通过 BGP 通告哪些 CIDR。定义要注入到 BGP 路由表中的前缀。
    • CiliumBGPNodeConfigOverride：定义节点特定的 BGP 配置，以提供更精细的控制。

    CiliumBGPPeeringPolicy 和 CiliumBGPClusterConfig 不应同时使用。如果这两个资源都存在，并且 Cilium 代理根据节点选择器与两者都匹配，则 CiliumBGPPeeringPolicy 将优先使用。

    # CiliumBGPClusterConfig

    CiliumBGPClusterConfig 资源用于根据其 nodeSelector 字段为集群中的一个或多个节点定义 BGP 配置。每个 CiliumBGPClusterConfig 定义一个或多个 BGP 实例，这些实例由其 name 字段唯一标识。

    一个 BGP 实例可以有一个或多个对等体。每个对等体都由其名称字段唯一标识。对等体的自治系统编号和地址分别由 peerASN 和 peerAddress 字段定义。对等体的配置由 peerConfigRef 字段定义，该字段是对对等体配置资源的引用。

    peerConfigRef 中的 group 和 kind 是可选的，默认值分别为 cilium.io 和 CiliumBGPPeerConfig。

    默认情况下，BGP 控制平面实例化的每个路由器实例都没有监听端口。这意味着 BGP 路由器只能发起与已配置对等体的连接，而不能接受传入连接。这是默认行为，因为 BGP 控制平面设计用于在同一节点上运行另一个 BGP 路由器（例如 Bird）的环境中。当需要接受传入连接时，可以使用 localPort 字段指定监听端口。

    监听默认 BGP 端口 (179) 需要 CAP_NET_BIND_SERVICE 权限。如果您希望使用默认端口，则必须授予 CAP_NET_BIND_SERVICE 权限，并在 Helm 配置中设置 securityContext.capabilities.ciliumAgent 的值。

    Cilium BGP 控制平面也支持 BGP 对等体的自动发现。 启用后，自动发现功能会自动配置 BGP 对等体的 IP 地址。具体地址的选择取决于启用的模式。 Cilium BGP 控制平面目前支持在 CiliumBGPClusterConfig 的 autoDiscovery 字段下使用 DefaultGateway 模式进行自动发现。

    默认网关自动发现

    默认网关自动发现模式允许 Cilium 自动发现并与指定地址族的默认网关（通常是机架顶部交换机）建立 BGP 会话。 要启用默认网关自动发现，请在对等体配置中配置 autoDiscovery 字段：

    peers:
- name: "tor-switch"
  peerASN: 65000
  autoDiscovery:
    mode: "DefaultGateway"
    defaultGateway:
      addressFamily: ipv6  # Can be "ipv4" or "ipv6"
  peerConfigRef:
    name: "cilium-peer"

    以下是 ToR 交换机的 BGP 配置要求： ToR 交换机必须配置“bgp listen range”以支持动态 BGP 邻居。此配置使 ToR 交换机能够通过监听特定 IP 前缀范围内的连接来接受来自 Cilium 节点的 BGP 会话，从而无需知道每个 Cilium 节点的确切对等地址。 更多详情，请参阅 FRR 文档 - https://docs.frrouting.org/en/latest/bgp.html#clicmd-bgp-listen-range-A.B.C.D-M-X-X-X-X-M-peer-group-PGNAME

    为确保所有集群节点的 Cilium 配置保持一致，请为每个 ToR 交换机配置相同的本地 ASN（自治系统编号）。

    1
2
3
4

    router bgp 65100
  neighbor CILIUM peer-group
  neighbor CILIUM local-as 65000 no-prepend replace-as
  bgp listen range fd00:10:0:1::/64 peer-group CILIUM

    应用此配置后： Cilium 会确定每个节点上指定地址族的默认网关 它会自动与发现的网关建立 BGP 会话 它使用 peerConfigRef 引用的对等体配置作为会话参数

    不支持将链路本地地址用作默认网关。

    支持默认网关自动发现的多宿主

    在多宿主配置中，Cilium 节点连接到两个不同的机架顶部交换机。它会发现两个默认网关，但会选择 metric 度量值较低的默认路由来建立 BGP 会话。需要注意的是，Cilium 每次只为每个地址族创建一个 BGP 会话。如果度量值较低的默认路由发生故障或更改，则会触发协调机制，以与另一条默认路由的默认网关建立 BGP 会话。

    # CiliumBGPPeerConfig

    CiliumBGPPeerConfig 资源用于定义 BGP 对等体配置。多个对等体可以共享相同的配置，并引用同一个 CiliumBGPPeerConfig 资源。

    • Timers

    # MD5 Password

    CiliumBGPPeerConfig 中的 AuthSecretRef 可用于在与引用此配置的 BGP 对等体的会话上配置 RFC-2385 TCP MD5 密码。

    AuthSecretRef 应引用 BGP 密钥命名空间中的密钥名称（如果使用 Helm Chart，则默认为 kube-system）。该密钥应包含一个 key 为 password 。 BGP 密钥仅限于已配置的命名空间，以最大限度地减少每个 Cilium Agent 实例所需的权限。Helm Chart 会配置 Cilium 以使其能够默认读取该命名空间中的密钥。

    创建 secret 的例子： kubectl create secret generic -n kube-system --type=string secretname --from-literal=password=my-secret-password

    如果要更改命名空间，可以设置 Helm Chart 中的 bgpControlPlane.secretNamespace.name 值。要自动创建命名空间，可以将 Helm Chart 中的 bgpControlPlane.secretNamespace.create 值设置为 true。

    由于 TCP MD5 密码会对数据包的头部进行签名，因此如果会话由 Cilium 进行地址转换（换句话说，Cilium Agent 的 Pod IP 地址必须是 BGP 对等体看到的地址），则无法使用 TCP MD5 密码。 如果密码错误，或者头部被更改，则 TCP 连接将失败。Cilium Agent 的日志中将显示“dial: i/o timeout”，而不是更具体的错误消息。

    如果部署的 CiliumBGPPeerConfig 中包含 Cilium 找不到的 authSecretRef，则 BGP 会话将使用空密码，并且代理将记录如下示例中的错误 level=error msg="Failed to fetch secret \"secretname\": not found (will continue with empty password)" component=manager.fetchPeerPassword subsys=bgp-control-plane

kind: CiliumBGPPeerConfig
metadata:
  name: cilium-peer

    holdTimeSeconds: 9

    在部署了 Kubernetes 集群的数据中心网络中，通常建议将 HoldTimer 和 KeepaliveTimer 的值设置得较低，以便更快地检测到可能的故障。例如，您可以将 holdTimeSeconds 设置为最小值 9，keepAliveTimeSeconds 设置为 3。 为了确保在与对等节点失去连接后能够快速重新连接，请降低 connectRetryTimeSeconds 的值（例如，设置为 5 或更低）。由于内部会对配置值应用随机抖动，因此 ConnectRetryTimer 的实际值会在 [ConnectRetryTimeSeconds, 2 * ConnectRetryTimeSeconds) 区间内。

    # EBGP Multihop

    在 eBGP 中，BGP 数据包的 IP TTL 默认设置为 1。通常建议不要更改 TTL 值，但在某些情况下，您可能需要更改 TTL 值。例如，当 BGP 对等体是路由服务器且位于不同的子网时，您可能需要将 TTL 值设置为大于 1 的值。

    # Graceful Restart

    Cilium BGP 控制平面可以配置为优雅重启的 “Restarting Speaker”。启用优雅重启后，BGP 会话将重启，并且“优雅重启”功能会在 BGP OPEN 消息中通告。

    如果 Cilium 代理重启，对等 BGP 路由器不会立即撤回从 Cilium BGP 控制平面接收的路由。数据路径会在代理重启期间继续转发流量，因此不会造成流量中断。

    还可以选择使用 restartTimeSeconds 参数。restartTime 是通告给对等体的时长，Cilium BGP 控制平面预计在此时长内重新建立 BGP 会话。当 restartTime 到期时，对等体将移除之前由 Cilium BGP 控制平面通告的路由。

    1
2

    apiVersion: cilium.io/v2
kind: CiliumBGPPeerConfig
metadata:
  name: cilium-peer
spec:
  gracefulRestart:
    enabled: true
    restartTimeSeconds: 15

    当 Cilium Agent 重启时，它会关闭 BGP TCP 套接字，从而发出 TCP FIN 数据包。对端收到此 TCP FIN 数据包后，会将其 BGP 状态更改为 Idle，并启动其 RestartTime 定时器。 Cilium Agent 的启动时间因部署情况而异。如果使用 RestartTime，则应将其设置为大于 Cilium Agent 启动所需时间的时长。

    RestartTime 的默认值为 120 秒。

    # Transport

    CiliumBGPPeerConfig 的 Transport 可用于调整对等体 BGP 会话的连接设置。

    默认情况下，当 BGP 处于主动模式（Cilium 代理发起 TCP 连接）时，目标端口为 179，源端口为临时端口。可以使用 peerPort 字段配置自定义目标端口。

    默认情况下，BGP 会话的源 IP 地址会根据出口接口自动检测。可以使用 sourceInterface 字段覆盖此设置，指定网络接口中应用的 IP 地址。每个地址族中，接口必须只有一个非环回、非组播且非链路本地 IPv6 地址。

kind: CiliumBGPPeerConfig
metadata:
  name: cilium-peer
spec:
  transport:
    peerPort: 179
    sourceInterface: lo

    # Address Families

    “地址族”字段是一个包含 AFI（地址族标识符）、SAFI（后续地址族标识符）对和通告选择器的列表。目前仅支持 {afi: ipv4, safi: unicast} 和 {afi: ipv6, safi: unicast} 这两种 AFI/SAFI 选项。

    默认情况下，如果未指定地址族，BGP 控制平面会同时向对等体发送 IPv4 单播和 IPv6 单播多协议扩展能力 (RFC-4760)。

    在每个地址族中，您可以通过通告标签选择器控制路由发布。此处定义了各种通告类型。

    如果没有匹配的广告，则不会向对等方广播任何前缀。默认配置是不广播任何前缀。

    # BGP Advertisements

    CiliumBGPAdvertisement 资源用于定义各种通告类型及其关联属性。对等配置的 families 字段中定义的通告标签选择器可以与一个或多个 CiliumBGPAdvertisement 资源匹配。

    # BGP### Attributes属性

    您可以使用 advertises[*] 中的 attributes 字段，为 Cilium BGP 控制平面通告的前缀配置 BGP 路径属性。可以通告的路径属性有两种类型：团体属性 (Community) 和本地优先级 (LocalPreference)。

    以下是 CiliumBGPAdvertisement 资源的一个配置示例，该资源通告的 Pod 前缀的团体属性值为“65000:99”，本地优先级为 99。

      attributes:
        communities:
          standard: [ "65000:99" ]
        localPreference: 99

    # Local Preference

    LocalPreference 定义了在 BGP 本地优先级路径属性中通告的优先级值。由于本地优先级仅对 iBGP 对等体有效，因此该值对于 eBGP 对等体将被忽略（不会通告本地优先级路径属性）。
