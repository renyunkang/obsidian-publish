    # 1. 编译 ebpf - 调试

    # 遇到的问题&解决

    问题： make build 编译各个二进制，编译 bpf 会报错，相关类型转换的错误 同时运行 daemon cilium-agent 失败，无法编译 bpf 程序

    1

    error: implicit conversion loses integer precision: 'const __u64' (aka 'const unsigned long long') to '__u32' (aka 'unsigned int') [-Werror,-Wshorten-64-to-32]，

    去掉编译中的参数 -Werror 后，编译成功，使用 tc filter add 到网卡上时提示：

    1
2
3
4

    tc filter add dev lxcb0593b277315 ingress bpf da obj test.o sec tc
libbpf: BTF is required, but is missing or corrupted.
ERROR: opening BPF object file failed
Unable to load program

    注释掉所有的 lib/xxx.h 之后重新编译，便可以 tc filter add 成功

    解决： 需要使用下面的 clang 以及 llvm 进行编译 https://github.com/llvm/llvm-project.git

    1
2
3
4

    clang version 17.0.6 (https://github.com/llvm/llvm-project.git 6009708b4367171ccdbf4b5905cb6a803753fe18)
Target: unknown
Thread model: posix
InstalledDir: /usr/local/bin

    【注】：

    • 镜像中存放 bpf 源代码路径: /var/lib/cilium/bpf/ - （类似 make install 后的路径）
    • 编译后的存放路径为：/run/cilium/state - /var/run/cilium/state 软连接

    # 正常安装

    • make kind - 安装一个 kind 集群
    • make kind-image - 构建本地镜像并将需要的本地镜像拷贝到集群中
    • make kind-install-cilium - 使用 cilium-cli 安装 cilium
    • make kind-down - 删除 kind 集群

    # 调试安装

    • make kind-debug - 安装一个 kind 集群 + 构建带有调试信息的镜像并拷贝到集群中 + 使用cilium-cli 安装 cilium

    • make kind-debug-agent - 安装一个 kind 集群 + 构建带有调试信息的镜像(只有agent)并拷贝到集群中 + 使用cilium-cli 安装 cilium
