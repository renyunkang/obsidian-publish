    # Kube-proxy

    Kube-proxy 是 Kubernetes 中的网络组件，作为 daemonset 安装在集群中的每个节点上，负责 Service 和 Pod 之间的通信。其主要作用是维护 service-to-pod map的网络规则，包括必要时的NAT转换，以确保数据包到达预期目的地。Kube-proxy 利用 iptables 或 IPVS 充当 L3/L4 网络代理和负载均衡器，其中 iptables 作为开箱即用的默认设置。

    iptables

     Iptables 作为 Linux 内核中的数据包过滤器和防火墙工具。它根据一组基于 IP 地址或基于端口的规则 (TCP/UDP) 的用户定义规则来分析、修改和过滤网络数据包。

    iptables 虽然是一项可靠的技术，但其设计目的并不是为了处理现代 Kubernetes 部署的动态且广泛的网络需求。

    # eBPF - the Standard for Kubernetes Networking

    它允许直接在 Linux 内核中进行可编程处理，从而以高效的速度和灵活性实现广泛的网络、可观察性和安全相关任务

    对于负载平衡，iptables 从来都不是为高度扩展的操作而设计的。顺序规则匹配和严格的基于 IP 的规则很难应对频繁变化的 IP 地址；而 eBPF 使用高效的哈希表，允许几乎无限的规模。eBPF 接近 O(log n) 时间复杂度

    # Kube-proxy replacement

    将 kube-proxy iptables 替换为 ebpf — cilium，目前唯一毕业的 cni 项目。Cilium 在每个 Kubernetes 节点上安装 eBPF 和 XDP程序，绕过 iptables 的开销

    向 eBPF 的过渡不仅仅是技术升级；这是一个战略举措。 eBPF 在处理大规模操作（例如负载均衡）方面的效率已经变得显而易见。通过使用高效的哈希表，eBPF 几乎可以无限扩展，从而减少延迟和开销。

    改进的应用程序性能、简化的网络操作和增强的安全性可转化为成本节约和更好的资源利用率。减少延迟直接影响用户体验。
