    # 添加/验证脚本

    行中的配置 bgpfilter 目前仅适用了 action、cidr、matchOperator 三个字段，并且目前只考虑增加和验证操作，基于此需求提供一个简单的 shell 脚本辅助用户进行 bgpfilter 资源的变更操作

    此脚本需要 yq 和 grepcidr。

    在运行此脚本前，请确保您的环境中已安装以下工具：

    1. kubectl: 用于与您的 Kubernetes 集群进行交互。
    2. yq (v4+): 一个功能强大的命令行 YAML 处理器。
      • macOS (使用 Homebrew):
        1

        brew install yq
      • Linux (下载二进制文件):
        1

        sudo wget https://github.com/mikefarah/yq/releases/latest/download/yq_linux_amd64 -O /usr/local/bin/yq && sudo chmod +x /usr/local/bin/yq
    3. grepcidr: 一个用于 IP 地址和 CIDR 计算的工具。
      • macOS (使用 Homebrew):
        1

        brew install grepcidr
      • Debian/Ubuntu:
        1

        sudo apt-get update && sudo apt-get install grepcidr
      • CentOS/RHEL:
        1

        sudo yum install grepcidr

    Usage:
  ./bgpfilter.sh add --name=<bgpfilter-name> --cidr=<cidr1>[,<cidr2>,...] [--output=<output-file>]
  ./bgpfilter.sh verify --name=<bgpfilter-name>
  ./bgpfilter.sh verify --input=<input-file>

    add 操作 此命令用于更新一个已存在的 BGPFilter 资源，向其 importV4 和 exportV4 列表中同时添加新的 CIDR 规则。

    用法:

    1

    ./bgpfilter_pure.sh add --name=<bgpfilter-name> --cidr=<cidr1>[,<cidr2>,...] [--output=<output-file>]

    参数:

    • --name=<bgpfilter-name>: 必需。 指定要更新的 BGPFilter 资源的名称。如果资源不存在，脚本会报错退出。
    • --cidr=<cidr1>[,...]: 必需。 指定要添加的一个或多个 CIDR 网段，以逗号分隔。
    • --output=<output-file>: 可选。 如果提供此参数，更新后的 YAML 配置将输出到指定文件，而不是直接应用到集群中。

    行为特性:

    • 预验证: 在执行任何修改前，会先对获取到的 BGPFilter 资源进行一次完整的格式验证。
    • CIDR 容错:
      • 跳过无效 CIDR: 如果输入了格式错误的 CIDR，会提示并跳过。
      • 跳过已存在 CIDR: 如果要添加的 CIDR 规则已存在于 importV4 或 exportV4 中，会提示并跳过。
    • 输入去重: 自动对 --cidr 参数中提供的列表进行去重。
    • 重叠警告: 在添加前会检查新 CIDR 之间、以及新 CIDR 与目标资源中现有 CIDR 的重叠情况，并以非致命警告的形式输出。
    • 只支持 add 与 verify 操作
    • 增加操作 - add
      • name 指定添加的 bgpfilter name
      • cidr 指定要添加的网段，当有多个网段时使用逗号分割
      • output 指定添加后的输出到某个文件；如果未指定该参数会直接 apply 到集群中
    • 验证操作 - verify
      • name 指定要验证集群中的 bgpfilter
      • input 指定要验证指定文件中的 bgpfilter
      • name 和 input 同时只能指定其一

    verify 操作

    此命令用于对一个 BGPFilter 资源进行全面的静态检查。

    用法:

    1
2
3
4
5

    # 验证集群中的资源
./bgpfilter_pure.sh verify --name=<bgpfilter-name>

# 验证本地文件中的资源
./bgpfilter_pure.sh verify --input=<input-file>

    参数:

    • --name=<bgpfilter-name>: 指定要验证的、位于集群中的 BGPFilter 资源名称。
    • --input=<input-file>: 指定要验证的本地 YAML 文件路径。
    • --name 和 --input 两个参数必须提供其一，且不能同时使用。

    验证逻辑: verify 命令会执行全面的检查，并一次性报告所有发现的问题。

    • 致命错误 (导致验证失败):
      • apiVersion 和 kind 不正确（应为 crd.projectcalico.org/v1 和 BGPFilter）。
      • 规则中缺少 action, cidr, matchOperator 任意一个必需字段。
      • cidr 字段的值不是合法的 CIDR 格式。
    • 非致命警告 (不影响验证结果):
      • 在同一个规则列表内部 (例如, importV4 内部) 存在相互重叠的 CIDR。检测不会在 importV4 和 exportV4 之间进行。

    错误提示示例: 当一个规则缺少字段时，错误提示会包含该规则的完整内容（以单行 JSON 格式），方便快速定位。
