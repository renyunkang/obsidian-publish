    # 2. IPPool 绑定：

    ns 绑定使用的注解为：

    1
2
3
4
5

    annotations:
  'cni.projectcalico.org/ipv4pools': '["default-ipv4-ippool"]'

annotations:
  'cni.projectcalico.org/ipv6pools': '["default-ipv6-ippool"]'

    正常绑定 创建多个 ipv4、ipv6 的 ippool

    • ippool 绑定到 namespace
      • 创建新的namespace，并在该 namespace 下创建负载，查看 podip
    • ippool 绑定到负载， 创建负载 podip

    如果绑定关系在负载和 namespace 上均有设置，则以负载上的为准

    绑定失败

    • 没有可用 ippool，或者通过 nodeSelector 为 !all() 进行限制
    1

    networkPlugin cni failed to set up pod "xxxx" network: no configured Calico pools for node nodex
    • 注解写错（ipv4 绑定 ipv6 的地址池）
    1

    networkPlugin cni failed to set up pod "xxxx" network: "10.233.64.0" isn't a IPv6 address

    1

    networkPlugin cni failed to set up pod "xxxx" network: "fd8f:e6b7:6ea9::" isn't a IPv4 address

    # 3. BGP 连接测试：

    • v4 bgp peer断了，不影响v6 bpg peer
    • v6 bgp peer断了，不影响v4 bgp peer （测试方式同理）

4 S root      1092   616  0  80   0 -   461 do_pol 13:57 ?        00:00:00 bird -R -s /var/run/calico/bird.ctl -d -c /etc/calico/confd/config/bird.cfg
4 S root      1093   619  0  80   0 -   465 do_pol 13:57 ?        00:00:00 bird6 -R -s /var/run/calico/bird6.ctl -d -c /etc/calico/confd/config/bird6.cfg

    可以看到 bird 启动的命令。容器中 bird 启动时会根据配置文件进行启动、创建一个用于通信的 unix socket 文件，同时监听 179 端口用于建立 bgp 邻居。

    我们可以通过停掉 bird 进程，模拟意外退出。但是直接 kill 掉进程，可能会被 runsv bird 重新拉起来一个新的进程，因此我们可以通过删除其配置文件后 kill 掉 bird 进程。

    bgp 连接断掉后，各个 bgp peer 状态

    这个方法会触发 calico 健康异常，进而导致其自动重启，重启后配置文件会从镜像中重新生成，所以 bgp 连接会自动恢复，如果想长时间测试，可以拉长 calico 健康检查的时长。

    # 4. service 双栈IP：

    kubectl expose deploy netools，编辑 service netools 或者新建新的 service

    • 创建 svc，并指定 ipFamilyPolicy: PreferDualStack （当双栈不支持时，会退回单栈）
    • 创建 svc，并指定 ipFamilyPolicy: RequireDualStack

    新建/修改后查看 service yaml，查看 spec.clusterIPs 下已分配的 ipv6 的地址以及 ipFamilies 为 [IPv4, IPv6]

  - IPv4
  - IPv6

    targetPort: 80

    app: netools
  sessionAffinity: None
  type: ClusterIP

  loadBalancer: {}

    # 5. 跨节点联通性测试：

    • pod-pod 进入 pod 内部查看已分配的 ip，或者查看 pod yaml 中的注解获取 podip； 在 pod 中访问另一个节点的 podip
    • pod-node 查看调度到其他节点的 pod，检查相应的路由 ip -6 r，访问 podip
    • pod-service 进入 pod 内部使用 service name（域名） 访问 service 进入 pod 内部使用 service ip 访问 service
