    # calico ipam 相关 CRD 资源

    资源名介绍
    ippool用于 ipam 分配的 ip 池
    ipamblockblock 中 ip 分配信息等
    blockaffinities由 ippool 分割的细粒度的 ip 块以及 block 与节点的亲和性绑定关系
    ipamconfigs用于配置 ipam 相关参数
    ipamhandles用于保存 pod 与 block 关系

    apiVersion: projectcalico.org/v3
kind: IPPool
metadata:
  name: ippool-1
spec:
  blockSize: 24
  cidr: 10.233.0.0/18
  ipipMode: Always
  natOutgoing: true
  vxlanMode: Never
  disabled: false
  nodeSelector: all()
    字段说明取值默认值
    blockSize该 ippool 用于分配 ip 的 block 的 CIDR 大小。block 会按需分配给 node， 并用于聚条目。该合路由来减少路由值只能在创建 ippool 时设置ipv4：20~32 ipv6：116~128ipv4：26, ipv6：122
    cidrippool 的 ip 范围有效的 IPv4 或 IPv6 CIDR
    ipipMode使用 ipip 封装，不能与 vxlanMode 同时使用Always, CrossSubnet, NeverNever
    vxlanMode使用 vxlan 封装，不能与 ipipMode 同时使用Always, CrossSubnet, NeverNever
    natoutgoing启用后，ippool 中的容器发送到该 ippool 外目的地的数据包将被伪装true, falsefalse
    disabled当设置为 true 时，Calico IPAM 将不会从该池中分配地址true, falsefalse
    nodeSelector节点选择器，IPAM 允许该 ippool 的地址分配给的节点all()
    关于nodeSelector 取值应为表达式，符合 IP pool (tigera.io) 中的规范。

    # 分配流程

    创建 pod 并被 kube-controller-manager 将其调度到某一个节点之后，kubelet 创建真正的 pod 并调用 calico cni 分配 ip 并完成网络配置。

    # 指定分配

    • 遍历集群内 enable 的 ippool，并遍历查找包含指定 ip 地址的 ippool
    • 根据IP池的 blocksize 计算出IP地址对应的block cidr
    • 使用block cidr查询对应的block
      • 若查询结果为空，即 block 不存在，尝试创建 blockaffinity，并分配指定的IP。
      • 若查询到了对应的block，则尝试从block中分配指定的IP。
    • 分配指定的IP地址，首先会校验 block 与 node 的亲和性，如果设置了StrictAffinity参数，block必须与节点亲和才能够进行IP分配，否则会报错；之后进行分配，如果 IP 已被分配，则直接返回错误；如果此IP未分配，则进行分配并更新block的数据，创建 ipamhandles。

    # 自动分配

    • 确定 ippool 用户可能使用注解 cni.projectcalico.org/ipv4pools 指定分配 ip 的 ippool；如果用户没有指定则根据 ippool 的 nodeSelector 获取选择了当前调度节点的 ippool 此处存在一个问题：设置的 annotation 与 ippool 指定的 nodeSelector 不能同时工作 K8S Annotation cni.projectcalico.org/ipv4pools should respect nodeSelector · Issue #4656 · projectcalico/calico (github.com)
    • 获取亲和块并分配 获取符合所确定的 ip pool cidr 的 blockAffinities，并从其中 block 中尝试进行 ip 分配，如果没有合适的 block，根据 ipamconfigs 配置确定是否进行自动创建 block 并进行 ip 分配；如果无法再创建新的 block，根据 ipamconfigs 配置确定是否可以借用 ip 完成分配
    • 分配 IP并更新 ipamblock 的数据，创建 ipamhandles

    # 释放 ip

    根据分配时创建的 ipamhandles 获取 pod 所占用的 block，并完成释放 ip 操作。

    # 其他

    用于静态分配的 ippool 可以将其 ippool.nodeSelector 设置为 !all()，防止 ippool 被自动分配。与设置 disabled: true 不同，它仍然可以用于手动分配。由于 nodeSelector 不匹配任何节点，因此不会自动使用 IPPool。 IP pool (tigera.io)
