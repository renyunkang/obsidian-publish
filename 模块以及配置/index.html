    # 简单介绍

    Blackbox Exporter 是 Prometheus 社区提供的 官方黑盒监控解决方案,其允许用户通过: http\HTTPS\DNS\TCP\ICMP\gRPC的方式对网络进行探测.

    支持的协议：HTTP, HTTPS, DNS, TCP, ICMP, gRPC.

    各个模块的配置： blackbox_exporter/CONFIGURATION.md at master · prometheus/blackbox_exporter (github.com)

    示例配置： https://raw.githubusercontent.com/prometheus/blackbox_exporter/master/example.yml

    # 暴露/采集的 metrics

    1. 通用的 metrics scrape采集
    • scrape_duration_seconds
    • scrape_samples_post_metric_relabeling
    • scrape_samples_scraped
    • scrape_series_added

    probe 探测

    • up 探测是否启用
    • probe_success 是否探测成功
    • probe_ip_protocol 4 指定探测的 ip 协议 ipv4/ipv6
    • probe_dns_lookup_time_seconds DNS 查找所花费的时间（以秒为单位）
    • probe_duration_seconds 探测完成所花费的时间（以秒为单位）
    • probe_ip_addr_hash 指定 IP 地址的哈希值。 可用于检测 IP 地址是否更改

    1. icmp

      metrics取值解释
      probe_icmp_duration_seconds{phase=“resolve”}0.097182057icmp 请求的持续时间 （DNS 解析）
      probe_icmp_duration_seconds{phase=“rtt”}0.032648941icmp 请求的持续时间 （回路响应时间）
      probe_icmp_duration_seconds{phase=“setup”}7.1581e-05icmp 请求的持续时间 （setup）
      probe_icmp_reply_hop_limit46回复的数据包跳数限制（ipv4 的 TTL）

    2. tcp

      metrics取值解释
      probe_failed_due_to_regex0是否因为表达式探测失败

    3. http

      metrics取值解释
      probe_failed_due_to_regex0是否因为表达式探测失败
      probe_http_content_length-1http content 响应的长度
      probe_http_duration_seconds{phase=“connect”}0.011417546connect 阶段的 http 请求持续时间，对所有重定向求和
      probe_http_duration_seconds{phase=“processing”}0.025588187processing 阶段的 http 请求持续时间，对所有重定向求和
      probe_http_duration_seconds{phase=“resolve”}0.011763045resolve 阶段的 http 请求持续时间，对所有重定向求和
      probe_http_duration_seconds{phase=“tls”}0tls 握手阶段的 http 请求持续时间，对所有重定向求和
      probe_http_duration_seconds{phase=“transfer”}0.120212515transfer阶段的 http 请求持续时间，对所有重定向求和
      probe_http_redirects0重定向次数
      probe_http_ssl0最终重定向是否用 SSL
      probe_http_status_code200响应 HTTP 状态码
      probe_http_uncompressed_body_length370317未压缩响应 body 的长度
      probe_http_version1.1探测响应的 HTTP 版本

    4. dns

      metrics取值解释
      probe_dns_additional_rrs additional0资源记录列表中的条目数
      probe_dns_answer_rrs answer0资源记录列表中的条目数
      probe_dns_authority_rrs0规范资源记录列表中的条目数
      probe_dns_duration_seconds{phase=“connect”}0.000340141connect 阶段 DNS 请求的持续时间
      probe_dns_duration_seconds{phase=“request”}0.000257105request 阶段 DNS 请求的持续时间
      probe_dns_duration_seconds{phase=“resolve”}0.002221205resolve 阶段 DNS 请求的持续时间

    5. grpc

    # 小结

    blockbox 支持 http\HTTPS\DNS\TCP\ICMP\gRPC 的方式对网络进行探测， https 是通过 http 配置其他参数实现的。同时也可以通过配置其他参数来实现自定义探测。 针对光大对于 calico 监控的需求可以通过以下方面实现部分需求：

    • icmp 检测节点的联通性
    • tcp 检测 bgp 179 端口是否在线
