    # Overlay 部署

    默认安装，对底层网络基础设施的要求最低。cilium 支持基于 udp 封装的 vxlan 和 geneve 隧道。所有 cilium 节点间的流量都会被封装

    • vxlan（默认） UDP 端口为 8472；
    • Geneve UDP 端口为 6081

    优势：

    • 简单
    • 地址空间
    • 自动配置

    # Underlay 部署

    启用原生路由数据路径时，会设置 routing-mode: native，并启用原生数据包转发模式。原生数据包转发模式利用 Cilium 运行所在网络的路由功能，而不是进行数据包封装。

    在原生路由模式下，Cilium 会将所有未发往其他本地端点的数据包委托给 Linux 内核的路由子系统。这意味着数据包的路由方式与本地进程发出数据包的方式相同。因此，连接集群节点的网络必须能够路由 PodCIDR。

    依赖：

    • 为了运行原生路由模式，连接运行 Cilium 的主机的网络必须能够使用分配给 pod 或其他工作负载的地址转发 IP 流量。
    • 节点上的 Linux 内核必须知道如何转发运行 Cilium 的所有节点的 Pod 或其他工作负载的数据包。这可以通过两种方式实现：
      • 节点本身并不知道如何路由所有 Pod 的 IP 地址，但网络中存在一个路由器，它知道如何访问所有其他 Pod。在这种情况下，Linux 节点配置了一条默认路由，指向该路由器。此模型用于云提供商网络集成。
      • 每个节点都会获知所有其他节点的 Pod IP 地址，并将相应的路由信息​​添加到 Linux 内核路由表中。如果所有节点共享同一个 L2 网络，则可以通过启用 auto-direct-node-routes: true 选项来实现。否则，需要运行额外的系统组件（例如 BGP 守护进程）来分发路由。

    配置： 之前版本：

    • tunnel: disabled
    • ipv4-native-routing-cidr: x.x.x.x/y 1.9 之前 enable-endpoint-routes

    # 跨不同网段

    • native routing
    • autoDirectNodeRoutes=true：基于网络拓扑检测的路由，依赖于 ARP 解析和邻居发现。Cilium 会尝试通过检查节点是否存在于同一广播域来确定其是否可直接访问。如果 ARP 解析成功且无需网关，则该节点被视为本地节点，并安装直接路由。否则，该节点被视为远程节点，必须采用其他路由机制。
    • BGP + native routing

    bgp 是需要和所有的节点都建立 bgp 连接吗？是否可以只选取特定节点？ 类似 calico rr ？

    部署建议：

    1. 同子网 → Native Routing + eBPF Host-Routing（性能最优）。
    2. 跨子网 + 可控网络 → Native Routing (BGP) + eBPF。
    3. 跨子网 + 不可控网络（如云）→ VXLAN + eBPF。
