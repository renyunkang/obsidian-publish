# 需求整理

  1. 网络监控的自证，从监控和日志上给证据。(带宽、吞吐率、延迟、抖动、错误的监控)
  2. 网络错误后、业务的快速恢复。方案上可以切换集群、网络自修复等。(快速恢复环境的最佳实践文档)
  3. 文档细化
  • 文档 word 化，脑图 + 分析步骤
  • 脑图中一些说明补充为命令的指导(如四层的网络检测怎么做，nc命令的简要说明)
  1. 数据的采集、日志的采集(一键收集日志的脚本)
  2. [长期] 云上网络监控的整体规划，不依赖具体cni。
  • KSE 4.1 的网络可视化扩展组件
  • 巡检扩展组件里面针对网络的指标巡检
  • 部分组件添加sidecar的来做网络探测。

网络侧的快速响应-非业务侧通知 记录保留恢复后的模拟

apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  labels:
    control-plane: kube-proxy
  name: kube-proxy
  namespace: monitoring
spec:
  endpoints:
    - path: /metrics
      port: metrics
      bearerTokenFile: /var/run/secrets/kubernetes.io/serviceaccount/token
      tlsConfig:
        insecureSkipVerify: true
  namespaceSelector:
    matchNames:
      - kube-system
  selector:
    matchLabels:
      k8s-app: kube-proxy
