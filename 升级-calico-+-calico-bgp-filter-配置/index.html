    # 升级 calico

    官方文档：  Upgrade Calico on Kubernetes | Calico Documentation (tigera.io)

    对于 kk 安装的集群，默认使用 calico manifest 安装，因此依照官方文档的使用 manifest 的方案升级： 

    • https://docs.tigera.io/calico/latest/operations/upgrading/kubernetes-upgrade#upgrading-an-installation-that-uses-manifests-and-the-kubernetes-api-datastore

    # 1. 下载相应升级版本的 manifest 文件并编辑相关的配置

    节点数少于 50 的 manfiest：

    • curl https://raw.githubusercontent.com/projectcalico/calico/v3.26.1/manifests/calico.yaml -o upgrade.yaml 节点数大于 50 的，需要/已经安装 typha 的 manifest：
    • curl https://raw.githubusercontent.com/projectcalico/calico/v3.26.1/manifests/calico-typha.yaml -o upgrade.yaml

    根据需要编辑 upgrade.yaml 中 calico-daemonset 的环境变量配置：

    • ippool cidr
    • 是否启用 default ippool
    • ippool 默认模式选择(ipipmode)
    • 是否开启 natOutgoing
    • … …

    即修改对应 upgrade.yaml 文件中 calico-node daemonset container 的环境变量，下面的环境变量主要针对与默认的 default-ipv4-ippool 生效，如果客户对于该 ippool 有定制修改，即修改相应的环境变量（定制了哪些修改哪些）；没有额外的定制修改则修改 CALICO_IPV4POOL_CIDR 即可

    对于需要 typha 的还需要编辑 upgrade.yaml 中 calico-typha Deployment 副本数量：

    • 建议每 200 个节点至少有 1 个副本，并且不超过 20 个副本。在生产中，我们建议至少三个副本，以减少滚动升级和故障的影响。副本数量应始终小于节点数量，否则滚动升级将停止。

    # 2. 滚动更新

    • kubectl apply –server-ide –force-conflicts -f upgrade.yaml

    # 3. 观察更新状态，并等待 calico-node pods 最终为 ready

    • watch kubectl get pods -n kube-system

    # 4. 删除旧的 calicoctl，下载安装新的 calicoctl

    如果有额外配置的需要同步配置（连接外部datastore）；替换相应的版本进行下载，如果路径不正确的话，可以到官方 github release 页面下载：  https://github.com/projectcalico/calico/releases

    • curl -L https://github.com/projectcalico/calico/releases/v3.26.1/download/calicoctl-linux-amd64 -o calicoctl

    查看 calicoctl 的安装以及配置：  https://docs.tigera.io/calico/latest/operations/calicoctl/install

    # 5. 使用 calicoctl 验证  

    • calicoctl version

kind: BGPPeer

  name: rack1-to-leaf1
spec:
  peerIP: 10.1.5.1
  asNumber: 65005
  filters:
  - rack-to-leaf-filter
  nodeSelector: rack == 'rack1'

apiVersion: crd.projectcalico.org/v1

metadata:
  name: rack2-to-leaf2

  peerIP: 10.1.8.1

  filters:
  - rack-to-leaf-filter
  nodeSelector: rack == 'rack2'

apiVersion: crd.projectcalico.org/v1
kind: BGPFilter
metadata:
  name: rack-to-leaf-filter
spec:
  importV4:
    - action: Reject
      matchOperator: NotIn
      cidr: 10.233.64.0/18

    apiVersion: crd.projectcalico.org/v1
kind: BGPFilter
metadata:
  name: my-filter
spec:
  exportV4:
    - action: Accept
      matchOperator: In
      cidr: 77.0.0.0/16
    - action: Reject
      matchOperator: NotIn
      cidr: 88.0.0.0/16
  importV4:
    - action: Reject
      matchOperator: NotIn
      cidr: 44.0.0.0/16
  exportV6:
    - action: Reject
      matchOperator: NotEqual
      cidr: 9000::0/64
  importV6:
    - action: Accept
      matchOperator: Equal
      cidr: 5000::0/64
    - action: Reject
      matchOperator: NotIn
      cidr: 5000::0/64
    • importV4: 表示导入到本机的路由过滤规则，当建立 RR/TOR 模式时可以在该字段下添加一些规则。
      • cidr: IPv4 range
      • matchOperator: 匹配路由的方法，取值：In, NotIn, Equal, NotEqual
      • action: 对匹配路由采取的行为，取值：Accept, Reject
    • exportV4: 表示将本机本身的路由导入到 bird 进行宣告的过滤规则
