    • L4 策略可以独立使用，也可以与 L3 策略（如 toCIDR、toEndpoints）结合使用。

    • L3 策略控制谁能和谁通信（基于 IP/CIDR/标签），而 L4 策略控制能使用哪些端口和协议（如 TCP 80、UDP 53）。

    • 默认行为：

      • 如果未定义任何 L4 策略，端点（Pod）可以自由使用所有端口和协议（包括 ICMP）。
      • 如果定义了 L4 策略，则仅允许显式指定的端口和协议，其他流量（包括 ICMP）默认拒绝，除非是已允许连接的关联流量（如 ICMP 不可达消息）。

    L4 策略通过 toPorts 字段 定义，适用于 入口（Ingress）和出口（Egress） 流量。

    # label 相关的第 4 层规则

    允许所有带有 role=frontend 标签的 endpoint 与所有带有 role=backend 标签的 endpoint 通信，但它们必须使用 TCP 协议在 80 端口上进行通信。

    带有其他标签的 endpoint 将无法与带有 role=backend 标签的 endpoint 通信，而带有 role=frontend 标签的 endpoint 也无法通过 80 端口以外的 endpoint 与 role=backend endpoint 通信。

    apiVersion: "cilium.io/v2"
kind: CiliumNetworkPolicy
metadata:
  name: "l4-rule"
spec:
  endpointSelector:
    matchLabels:
      role: backend
  ingress:
  - fromEndpoints:
    - matchLabels:
        role: frontend
    toPorts:
    - ports:
      - port: "80"
        protocol: TCP

    # CIDR依赖型 第4层规则

    允许所有带有 role=crawler 标签的 endpoint 与 CIDR 192.0.2.0/24 内的所有远程目标通信，但它们必须使用 TCP 协议的 80 端口进行通信。

    该策略不允许没有 role=crawler 标签的 endpoint 与 CIDR 192.0.2.0/24 内的目标通信。此外，带有 role=crawler 标签的 endpoint 也无法通过 80 端口以外的端口与 CIDR 192.0.2.0/24 内的目标通信。

      role: crawler
  egress:
  - toCIDR:
    - 192.0.2.0/24
    toPorts:
    - ports:
      - port: "80"
        protocol: TCP

    # 限制 ICMP/ICMPv6 类型

    ICMP 策略可以与三层策略一起指定，也可以独立指定。 它限制 endpoint 发送和/或接收特定 ICMP/ICMPv6 类型（支持类型（整数）和对应的驼峰式消息（字符串））数据包的能力。如果指定了 ICMP 策略，则四层和 ICMP 通信将被阻止，除非该通信与策略允许的连接相关。 可以使用 icmps 字段在入口和出口处指定 ICMP 策略。

    限制所有带有 app=myService 标签的 endpoint 只能使用类型为 8 的 ICMP 和消息为 EchoRequest 的 ICMPv6 向任何第 3 层目标发送数据包

  name: "icmp-rule"
spec:
  endpointSelector:
    matchLabels:
      app: myService
  egress:
  - icmps:
    - fields:
      - type: 8
        family: IPv4
      - type: EchoRequest
        family: IPv6

    # 限制 TLS 服务器名称指示(TLS SNI)

    TLS SNI策略实施需要启用L7代理。

    当多个网站托管在同一台服务器上并共享一个 IP 地址时，服务器名称指示 (SNI)（TLS 协议的扩展）可确保客户端收到其尝试访问的网站的正确 SSL 证书。SNI 允许在 TLS 握手期间指定网站的主机名或域名，而不是在 HTTP 连接建立后握手时才指定。

    Cilium 网络策略可以将 endpoint 建立 TLS 握手的能力限制为指定的 SNI 列表。SNI 策略始终在egress 级别配置，并且通常与端口策略一起设置。

    以下规则限制所有带有 app=myService 标签的 endpoint 只能与 one.one.one.one SNI 建立 TLS 连接。任何其他尝试与其它 SNI（例如 cilium.io）建立连接的努力都将被拒绝。

kind: CiliumNetworkPolicy

  name: "l4-sni-rule"

  endpointSelector:
    matchLabels:
      app: myService
  egress:
  - toPorts:
    - ports:
      - port: "443"
        protocol: TCP
      serverNames:
      - one.one.one.one
