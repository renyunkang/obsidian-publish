安装的组件

  • ds：
    • hybridnet-daemon
        • ipam
      • bgp 路由宣告
      • vlan + vxlan 打通
  • deploy： 全部使用主机网络
    • hybridnet-manager 控制面
    • hybridnet-webhook
      • ValidatingWebhookConfiguration
        • pods、remoteclusters、remotesubnets、networks、subnets
      • MutatingWebhookConfiguration
        • pods - 添加 annotation + nodeSelector
    • calico-typha
      • 网络策略 + 减少 apiserver 负载

模式

  • overlay 模式 - vxlan
    • Overlay = NetworkType(“Overlay”)
  • underlay 模式 - vlan or bgp
    • Underlay = NetworkType(“Underlay”)
    • GlobalBGP = NetworkType(“GlobalBGP”)

# 网络拓扑

hybirdnet 中有两个重要概念：网络域、子网

  1. 网络域 network - 实现了依赖特定网络资源 pod 的自动调度 一个网络域具有这样的性质：“一个带有特定地址的 Pod 如果能被部署到网络域内的某一节点，那么带有相同地址的 Pod 应该能被调度到同一网络域内的其他任意节点”。Network 对象通过 nodeSelector 选择带有特定 label 的节点作为网络域中的节点。
  2. Subnet - 描述 ip 地址分配信息

通过官方 [github 示例链接]( hybridnet/samples at main · alibaba/hybridnet (github.com))中可以创建不同模式的网络域以及子网，一个节点可以同时在多个网络域中,

# 存在的问题：

问题1：使用 kind 搭建 环境无法搭建起来 kind 创建的 node 节点没有/proc/sys/net/ipv4/neigh/default/gc_thresh1 文件。也不能在 kind 创建的节点容器内部创建 /proc/sys/net/ipv4/neigh/default/gc_thresh1 文件，因为 /proc 文件系统是一个虚拟文件系统，它用于在内核和用户空间之间传递信息。也不能在容器内部启用内核模块，因为容器共享宿主机的内核。 修改了源代码，注释掉相关部分后可以成功运行 使用修改后的镜像 docker pull rykren/hybridnet:release-amd64

  1. bgp 建立 peer 时 ttl 为1，因此建立 bgp peer 时不能跨多个交换机建立 peer

  2. 高于1.24版本的可能存在问题 hybridnet 在k8s1.26.3中使用异常 · Issue #372 · alibaba/hybridnet (github.com) 可以使用 --set daemon.enableFelixPolicy=false 不启用calico-typha即不启用网络隔离

1
2
3

It seems the version of felix we use (v3.20.2) is not compatible with higher version of kubernetes than 1.24, [projectcalico/calico#7032](https://github.com/projectcalico/calico/issues/7032).

We will fix it and you can deploy hybridnet without networkpolicy implementation by using the `--set daemon.enableFelixPolicy=false` helm parameter to avoid this in a v1.26.3 Kubernetes at present.
